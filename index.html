Streamlined Workflow

1

Connect XCF

Start with use xcf to activate the tool in your AI assistant.

use xcf
All xcf systems go!
2

Explore Projects

List available Xcode projects and select one to work with.

show
1. /Users/developer/Projects/MyApp.xcodeproj 2. /Users/developer/Projects/Library.xcodeproj
open 1
Current project: /Users/developer/Projects/MyApp.xcodeproj
3

View Code

Use the simple snippet command to view file contents.

snippet AppDelegate.swift
```swift import UIKit @main class AppDelegate: UIResponder, UIApplicationDelegate { // App delegate implementation } ```
4

Build Project

Build your project and get detailed error reports with code context.

build
File:`/Users/developer/Projects/MyApp/ViewController.swift`:34:22 [Error] Value of type 'String' has no member 'toInt' ```swift func processValue() { let value = "123" let number = value.toInt() // Error here } ```
5

AI Fixes Issues

Your AI assistant can now fix issues with precise context.

The error is occurring because Swift strings don't have a toInt() method. You should use Int(string) instead:

func processValue() {
    let value = "123"
    let number = Int(value)  // Fixed: proper Swift int conversion
}
6

Run Project

After fixing issues, run your project directly from the chat.

run

Setup Guide

1. Installation

git clone https://github.com/codefreezeai/xcf.git
cd xcf
xcodebuild -project xcf.xcodeproj -scheme xcf -configuration Release
‚úì

2. Configure Your AI Assistant

XCF is written in 100% Swift and integrates smoothly with the Apple ecosystem.

Cursor

Edit ~/.cursor/mcp.json:

‚úì 
{
  "mcpServers": {
    "xcf": {
      "type": "stdio",
      "command": "/path/to/xcf"
    }
  }
}

Claude Desktop

Edit ~/Library/Application Support/Claude/claude_desktop_config.json:

‚úì 
{
  "mcpServers": {
    "xcf": {
      "type": "stdio",
      "command": "/path/to/xcf",
      "args": [],
      "env": {
        "XCODE_PROJECT_FOLDER": "/path/to/project/",
        "XCODE_PROJECT": "/path/to/project/project.xcodeproj"
      }
    }
  }
}

3. Restart Your AI Assistant

Close and reopen your AI assistant application to apply the changes.

4. Grant Permissions

Use the command xcf grant to grant Xcode automation permissions.

Commands

Command Description
use xcf Activate xcf mode
show List open Xcode projects
open # Select project by number
run Run current project
build Build current project
current Show selected project
env Show environment variables
help Display all available commands
grant Grant permission to use Xcode automation

Function Calls

For programmatic access, XCF provides these MCP tools:

  • mcp_xcf_xcf: Execute xcf actions/commands
  • mcp_xcf_list: Show all available tools
  • mcp_xcf_snippet: Access file contents
  • mcp_xcf_help: Get help information

Using Snippets

XCF's snippet tool makes code viewing simple:

Human Commands

‚úì 
# View entire file with just filename
snippet AppDelegate.swift

# View specific lines (if needed)
snippet AppDelegate.swift 10 20

Function Calls

‚úì 
# Get entire file
mcp_xcf_snippet(filePath="AppDelegate.swift", entireFile=true)

# Get specific lines
mcp_xcf_snippet(filePath="AppDelegate.swift", startLine=10, endLine=20)

Security Features

XCF is designed with security in mind, providing:

Workspace protection Safely works with projects in your designated workspace
Boundary enforcement Automatically prevents access outside your workspace boundaries
Safe alternatives Redirects to secure options when needed
Environment controls Uses environment variables to define secure boundaries

Advanced Configuration

For non-Cursor clients or users who want more project-level control, you can add these optional environment variables:

‚úì 
{
  "mcpServers": {
    "xcf": {
      "type": "stdio",
      "command": "/path/to/xcf",
      "args": [],
      "env": {
        "XCODE_PROJECT_FOLDER": "/path/to/project/",
        "XCODE_PROJECT": "/path/to/project/project.xcodeproj"
      }
    }
  }
}

These environment variables let you:

Project selection Pre-select a specific Xcode project to work with
Security boundaries Define a custom workspace boundary for security

Troubleshooting

If you're having issues with XCF, check these common solutions:

Verify installation Make sure XCF is correctly installed
Check configuration Confirm your settings are valid in the configuration file
Permission issues Ensure permissions are properly set for Xcode access
Environment variables Use the env command to verify your environment setup
Restart your assistant Sometimes a simple restart will resolve connection issues
