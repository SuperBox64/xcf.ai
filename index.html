Setup Guide

INSTALLATION: Getting started with XCF is simple. You can install it directly from GitHub releases.

# Clone the repository
git clone https://github.com/codefreezeai/xcf.git
cd xcf

# Build the project
swift build -c release

# Create a symbolic link for easy access
ln -s "$(pwd)/.build/release/xcf" /usr/local/bin/xcf

PERMISSIONS: Grant XCF the permissions needed to automate Xcode:

# Grant Xcode automation permissions
xcf grant

FOR CURSOR USERS: Configure XCF as an MCP server by editing your ~/.cursor/mcp.json file:

{
  "mcpServers": {
    "xcf": {
      "type": "stdio",
      "command": "/Applications/xcf.app/Contents/MacOS/xcf"
    }
  }
}

FOR CLAUDE DESKTOP USERS: Add XCF to your claude_desktop_config.json file:

{
  "mcpServers": {
    "xcf": {
      "type": "stdio",
      "command": "/Applications/xcf.app/Contents/MacOS/xcf",
      "env": {
          "XCODE_PROJECT_FOLDER_optional": "/path/to/project/",
          "XCODE_PROJECT_optional": "/path/to/project/project.xcodeproj"
      }
    }
  }
}

RESTART YOUR AI ASSISTANT: After configuration, restart your AI assistant application to apply the changes and begin using XCF's powerful features.

TROUBLESHOOTING: If you encounter any issues with XCF, try these steps:

# 1. Quit your AI assistant
# 2. Open Terminal
# 3. Kill any running XCF processes `killall xcf`
# 4. Reopen your AI assistant
# 5. From AI: Activate XCF `use xcf`

PRECISE PROJECT SELECTION: XCF allows you to select specific Xcode projects through simple commands (projects must be within your designated workspace folder for security):

# List all available Xcode projects
xcf list

# Set the active project
xcf open #

# Run the active project
xcf run

SECURITY CONSIDERATIONS: XCF is designed with security in mind, providing workspace protection that safely works with projects in your designated workspace and boundary enforcement that automatically prevents access outside those boundaries.

The complete installation guide and documentation are available in our GitHub repository, providing detailed instructions for every step of the setup process.

XCF Command Reference

Explore the powerful commands available in XCF to enhance your Xcode development workflow

Core Commands

  • use xcf - Activate XCF mode to connect AI assistants to Xcode
  • xcf grant - Grant necessary permissions for Xcode automation
  • xcf show - List all available Xcode projects and workspaces
  • xcf open # - Select a specific Xcode project by number

Project Commands

  • xcf build - Build the currently selected Xcode project
  • xcf run - Execute the currently selected Xcode project
  • xcf current - Display information about the currently selected project
  • xcf env - Show all environment variables for the current context

Workflow Examples

Basic Development Workflow

  1. use xcf - Activate the XCF tool
  2. show - See all available Xcode projects
  3. open # - Select a specific project by number
  4. build - Build the selected project

Code Exploration

Additional Tools

  • snippet - Extract code snippets from files in the current project
  • list - Show all available tools in the XCF server
  • help - Display help information about XCF actions and usage

Note: You can access these commands after activating XCF with use xcf. After activation, you can use short commands like build instead of xcf build.

Code Snippets

XCF helps your AI assistant understand and fix code with powerful snippet extraction.

For AI Function Calls

To get an entire file:
mcp_xcf_snippet(filePath="/full/path/to/file.swift", entireFile=true)

For specific line ranges:
mcp_xcf_snippet(filePath="/full/path/to/file.swift", startLine=10, endLine=20)

For Human Commands

XCF now supports simplified, user-friendly snippet commands:

To get an entire file, just use the filename:
snippet filename.swift

For specific line ranges (still available but typically not needed):
snippet /full/path/to/file.swift 10 20

Smart Path Resolution

When a file isn't found at the exact path, XCF will intelligently search for it in:

  • The current working directory
  • The workspace folder specified in the environment
  • Subdirectories (one level deep) in the workspace
  • The current project directory

This means you can usually just use the filename without any path:

snippet Constants.swift

The simplified syntax makes code exploration much faster and more intuitive during conversations with your AI assistant.
